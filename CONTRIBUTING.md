# CONTRIBUTING

## Principes

- privilégier la documentation officielle Keycloak ;
- distinguer clairement :
  - supporté officiellement ;
  - recommandé ;
  - possible mais non recommandé ;
  - expérimental ;
- éviter les affirmations sans source ;
- documenter les hypothèses d’architecture.

## Style
- phrases simples ;
- sections stables ;
- exemples exécutables si possible ;
- README par lab ;
- références officielles en fin de document.

## Commits
Exemples :
- `docs: add support matrix for HA topologies`
- `labs: add benchmark baseline for k6`
- `runbooks: add login loop incident guide`
