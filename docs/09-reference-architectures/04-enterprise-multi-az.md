# Reference Architecture — Enterprise Multi-AZ

## Objectif
Tolérer une perte partielle sans rendre l’exploitation trop complexe.

## Briques
- répartition des pods ;
- dépendances réseau contrôlées ;
- base compatible avec l’objectif de résilience ;
- tests de perte de zone ou de nœud ;
- preuves de stabilité.
