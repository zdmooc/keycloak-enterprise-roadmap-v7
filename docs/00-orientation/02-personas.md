# Personas

## Objectif
Identifier les principaux profils visés par le dépôt.

## Quand utiliser
Quand on veut adapter le parcours à un usage précis.

## Prérequis
- compréhension des rôles IT entreprise

## Concepts clés
- administrateur IAM
- architecte sécurité
- architecte plateforme
- SRE / exploitant
- développeur intégrateur

## Procédure / design
Définir pour chaque persona : objectifs, sections prioritaires, labs utiles, livrables attendus.

## Pièges fréquents
- construire un parcours unique pour tous
- négliger l’angle exploitation

## Checklist
- [ ] 4 à 6 personas définis
- [ ] sections prioritaires par persona
- [ ] livrables associés
