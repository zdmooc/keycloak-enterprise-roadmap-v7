# How to use this repo

## Objectif
Expliquer comment consommer le dépôt selon le profil lecteur.

## Quand utiliser
Quand on veut utiliser le dépôt comme support d’apprentissage, de mission ou de build interne.

## Prérequis
- Git de base
- Markdown
- Docker ou Kubernetes selon le lab

## Concepts clés
- lecture séquentielle
- lecture par persona
- labs avec preuves

## Procédure / design
Choisir un persona, suivre le parcours correspondant, exécuter les labs dans l’ordre, déposer les preuves dans `evidence/`, puis ouvrir les runbooks et la gouvernance.

## Pièges fréquents
- ouvrir des labs sans avoir lu les prérequis
- confondre exemples de départ et design final

## Checklist
- [ ] Un parcours par persona est défini
- [ ] Les labs ont un README
- [ ] Les preuves attendues sont décrites
