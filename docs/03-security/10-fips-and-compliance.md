# FIPS and Compliance

## Objectif
Repérer les cas où la conformité impose des choix supplémentaires.

## Contenu minimal
- exigences réglementaires applicables ;
- contraintes cryptographiques ;
- dépendances de plateforme ;
- impacts sur les bibliothèques, certificats et algorithmes ;
- preuves attendues par audit.

## Règle de prudence
Ne pas activer un mode ou une posture de conformité “par principe” sans vérifier :
- le besoin réel ;
- le support de la plateforme ;
- l’impact sur les intégrations ;
- la méthode de preuve.
