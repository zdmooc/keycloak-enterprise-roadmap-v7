# Supported vs Preview Features

## Objectif
Éviter d’introduire une fonctionnalité parce qu’elle est séduisante sans vérifier sa maturité.

## Méthode
Pour chaque fonctionnalité :
- statut ;
- version d’apparition ;
- besoin métier ;
- impact sécurité ;
- impact run ;
- preuve en lab ;
- décision d’adoption ou d’attente.
