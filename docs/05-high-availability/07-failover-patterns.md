# Failover Patterns

## Objectif
Lister les scénarios de panne à tester réellement.

## Scénarios minimaux
- perte d’un pod ;
- redémarrage d’un pod ;
- perte d’un nœud ;
- indisponibilité transitoire DB ;
- erreur de configuration proxy ;
- upgrade interrompu.

## Pour chaque scénario
- impact attendu ;
- impact observé ;
- métriques ;
- durée ;
- décision d’amélioration.
