# OpenTelemetry and Log Correlation

## Objectif
Préparer une observabilité plus transverse.

## Démarche
- identifier les signaux déjà disponibles ;
- définir ce qui relève des logs, des métriques et du tracing ;
- corréler Keycloak avec reverse proxy, gateway, applications et base ;
- éviter l’instrumentation “pour la beauté du geste” sans usage opérationnel.
