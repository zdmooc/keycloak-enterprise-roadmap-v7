# Load Test Method

## Objectif
Tester la plateforme avec une méthode stable.

## Étapes
1. définir les scénarios ;
2. définir les volumes ;
3. définir la durée ;
4. collecter les métriques ;
5. comparer plusieurs runs ;
6. documenter les écarts.

## Scénarios de départ
- login interactif ;
- token request service account ;
- rafraîchissement ;
- lecture de métadonnées ;
- scénario avec panne injectée.
