# Runbook — Upgrade Precheck

## Objectif
Préparer une montée de version sans angle mort évident.

## Étapes
1. relever la version source et la cible ;
2. lire les release notes ;
3. lire le guide d’upgrade ;
4. exécuter `update-compatibility` ;
5. lister les providers, thèmes et features activées ;
6. définir rolling ou recreate ;
7. préparer les tests post-changement ;
8. fixer la stratégie de retour arrière.

## Preuves
- sortie des commandes ;
- captures ;
- décision formalisée.
